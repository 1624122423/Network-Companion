Non-Functional Requirements (draft)
Performance: The system should handle multiple users booking simultaneously without significant delays.
Security: User accounts, passwords, and personal information must be encrypted and stored securely.
Usability: The interface should be simple and intuitive for both mentors and mentees.
Reliability: The system should ensure that booking confirmations and notifications are delivered consistently.
Scalability: The system should support growth in the number of users and bookings without performance degradation.
