Functional Requirements:

User Accounts:

Users can create accounts with a username and password.

Use email for notifications and verification.

Support forgotten password and password reset functions.

Allow users to switch between Mentee Mode and Mentor Mode after signing up or through the profile page.

Profile:

Each profile includes a eamil adress linked to the account.

Display today’s available time slots for mentors in their profile.

Show location information (e.g., city, campus, or meeting place).

Include experience details such as industry category, years of work, and position.

Mentees can add interests during signup, which will later be matched to mentors based on their professional fields.

Time Slot Management (Mentor):

Mentors can define availability by adding time slots with location, start time, and end time.

Mentors can view, update, and cancel existing time slots.

All added slots are automatically displayed in the mentor’s calendar.


Booking (Mentee):

Mentees can choose from available mentor time slots for booking.

Mentees can send a short message or note when booking.

A confirmation email or message is sent upon successful booking.

Both mentors and mentees can cancel or reschedule bookings if necessary.


Search and Recommendation:

Mentees can search for mentors by name, keyword, or category.

Implement a recommendation system on the dashboard that suggests mentors based on:

Mentee’s selected interests

Mentor’s field or position

Frequency of previous bookings or mentor popularity



Dashboard:

Display recommended mentors prominently on the mentee dashboard.
Show upcoming bookings, notifications, and mentor availability updates.
Mentors can view their booked sessions and pending requests on their dashboard.
