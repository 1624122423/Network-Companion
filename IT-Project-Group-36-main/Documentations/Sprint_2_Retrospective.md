
# Sprint 2 Retrospective

During Sprint 2, our team mainly focused on improving project organization and technical foundations based on feedback from Progress Report 1. We made strong progress in defining our backend architecture, task management, and collaboration workflow.

## What went well
1. Established a clear GitHub branching strategy — each feature has its own branch, merged into `dev` and `main` after testing, reducing merge conflicts.
2. Adopted Jira as our Kanban tool to record sprint progress and align with weekly stand-ups.
3. Finalized backend stack: **MongoEngine (NoSQL)** for flexible data modeling and **Flask Session** for user authentication.
4. Improved documentation and recorded detailed meeting notes; completed Use Case and Low-/High-Fidelity designs.

## What could be improved
1. Some early diagrams were incomplete and lacked user–database interaction details.
2. Tool justification and coding conventions needed better documentation.
3. Stand-up meetings were not structured enough and sometimes lacked focus.

## Action items for next sprint
1. Further refine Jira tasks and assign owners for each item.
2. Apply consistent coding standards for naming, comments, and documentation.
3. Hold more frequent, structured stand-ups following “What I did / am doing / will do.”
4. Begin initial backend implementation (focusing on authentication and booking modules).
