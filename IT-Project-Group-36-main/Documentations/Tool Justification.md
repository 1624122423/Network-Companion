Tool Justification 
For Networking Companion Project Group36
1. Frontend Tool: React + Vite
React combined with Vite is chosen for the frontend development of this project. React provides a component-based architecture, making it easier to build reusable and maintainable UI elements. It has a large ecosystem and strong community support, which ensures stability and access to numerous third-party libraries. Vite complements React by providing a fast development server and an optimized build process. Since most of the team members do not have prior web development experience, React was selected as it is relatively beginner-friendly and has extensive documentation and learning resources.
2. Backend Tool: Python
Python is selected as the backend language because of its simplicity, readability, and wide usage in web development. It supports frameworks such as Flask or FastAPI, which make it easy to design RESTful APIs that integrate smoothly with the React frontend. Python also has extensive libraries for handling data, authentication, and testing, which helps reduce development effort and ensures reliability.
3. Database Tool: MongoDB
MongoDB is chosen as the database solution due to its flexibility as a NoSQL, document-oriented database. It stores data in JSON-like documents, which align naturally with the API structures used in the backend. This makes data exchange between the backend and database efficient. Additionally, MongoDB scales easily and can handle unstructured or semi-structured data, which allows future project growth.
4. Version Control: GitHub
GitHub is selected as the version control and collaboration platform. It provides distributed version control through Git, which ensures that every team member has a complete local copy of the repository. GitHub also facilitates pull requests, code reviews, and branch management. This workflow aligns with agile development, enabling effective collaboration, issue tracking, and integration with other tools.
5. Documentation & Reporting Tools
For documentation during development, Google Drive will be used. It allows the team to collaborate in real time, track each other’s contributions, and manage drafts effectively. The ability to comment and suggest edits in Google Docs ensures that communication and feedback can be captured efficiently.

For the final report, Confluence is tentatively chosen. The reason for this is that Confluence provides a strong pre-structured environment for formal documentation, which is well suited for the final deliverable. This combination allows the team to use Google Drive for day-to-day flexibility and Confluence for producing the final structured report.
6. Authentication Tool: Firebase
Firebase Authentication is chosen to handle user login and identity management. It supports multiple authentication methods such as email/password, Google sign-in, and other social providers out-of-the-box. This reduces development time compared to building a custom authentication system. Firebase also provides secure token-based authentication that integrates smoothly with the backend, ensuring user data protection and scalability.
7. Job Assignment Tool: Kanban Board
A Kanban board is used for job allocation and sprint planning. The visual representation of tasks in 'To Do', 'In Progress', and 'Done' columns allows the team to clearly track workload and progress. It also promotes transparency and accountability, as every team member can see who is working on what. Kanban fits well with agile practices and helps ensure smooth sprint execution and continuous delivery.
