
# Sprint 3 Retrospective

During Sprint 3, the team focused on implementing improvements identified in Sprint 2, particularly enhancing communication, task allocation, documentation quality, and presentation preparation. Significant progress was achieved in backend development and team coordination, ensuring readiness for the final presentation.

## What went well
1. Successfully implemented Sprint 2 action items: more frequent meetings, refined task breakdowns, and clearer role assignments.
2. Utilized Jira to manage detailed user-story-based tasks with assigned owners.
3. Improved documentation by fixing broken links and completing Hi-fi and Low-fi designs.
4. Configured GitHub Actions for automated testing after each main branch merge.
5. Conducted multiple presentation rehearsals, ensuring timing and delivery met requirements.

## What could be improved
1. Team communication still needs enhancement; some progress updates were delayed.
2. Inconsistent adherence to established coding conventions among a few members.
3. Testing documentation and manual test procedures require further development.

## Action items for next sprint
1. Maintain the current three-day meeting cycle to sustain active communication.
2. Expand test coverage and clarify manual testing workflows.
3. After the presentation, focus on full-system integration and user testing to ensure functionality and data consistency.
