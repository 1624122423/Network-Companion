🎓 Usage Scenarios
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Scenario 1: Quick Deployment (Beginners)
  1. Download: docker-compose.yml + Dockerfiles + nginx.conf + .env.example
  2. Download: deploy.sh + QUICKSTART.md
  3. Run: ./deploy.sh
  
Scenario 2: Production Deployment (Recommended)
  1. Download all configuration files
  2. Read README.md and ADDING_FILES_GUIDE.md
  3. Configure environment and security settings per guide
  4. Manual deploy: docker-compose up --build -d

Scenario 3: Local Development
  1. Use docker-compose.local-mongo.yml
  2. Optional: Replace with app_factory.py.new
  3. Start: docker-compose -f docker-compose.local-mongo.yml up -d


✅ Pre-Deployment Checklist
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

System Requirements:
  □ Docker installed (20.x+)
  □ Docker Compose installed (2.x+)
  □ At least 8GB available RAM
  □ Ports 80 and 5000 not in use

File Preparation:
  □ All config files in project root
  □ flask-server/it-project-auth.json exists
  □ Created .env file (from .env.example)
  □ deploy.sh has execute permission
